
### 2. 八条铁律

> **以下规则不可违反、不可简化、不可跳过。**

| # | 铁律 | 说明 |
|---|------|------|
| 1 | **严格顺序** | 按 `task-checklist.md` 中 T-0.1 → T-10.4 的编号顺序逐项执行，**严禁跳步、并行、乱序** |
| 2 | **先测后过** | 每项任务完成后**必须执行该项的全部测试**。所有测试通过后方可进入下一项 |
| 3 | **失败即停** | 任何一项测试未通过，**立即停止前进**。执行该项的回退操作，修复后重新测试 |
| 4 | **回退优先** | 修复时优先使用 `task-checklist.md` 中标注的回退路径，而非自行发明修复方案 |
| 5 | **编译即验** | 凡涉及代码变更的任务，`npm run build` 必须零错误零警告才算编译通过 |
| 6 | **需求为准** | 实现细节有疑问时，**回查 `requirements.md` 原文**，以需求文档为唯一准确定义 |
| 7 | **不做多余** | 不添加需求文档之外的功能、不引入计划之外的依赖、不修改与当前任务无关的文件 |
| 8 | **自主闭环** | 全程不向人类提问、不等待人类确认。遇到技术障碍自行查阅文档或尝试替代方案解决 |

### 3. 质量门槛

每一项任务你都必须确保以下质量标准**全部满足**后才能标记通过：

#### 代码质量门
- TypeScript 编译零错误（`npm run build` 或 `npx tsc --noEmit`）
- 无未使用的导入和变量
- 所有 API 返回统一格式 `{ code, data/message }`

#### 功能质量门
- 任务清单中列出的**每一条测试**都已实际执行并通过
- 不是"应该可以工作"，而是"已验证确实工作"

#### 安全质量门（适用于后端任务）
- 所有管理接口都有 JWT 认证中间件保护
- 密码使用 bcrypt 哈希，绝不明文存储
- 用户输入有校验，SQL 通过 Prisma 参数化

#### 兼容性质量门（适用于前端任务）
- 桌面端（≥768px）和移动端（<768px）都正常
- 中英文切换后界面完整无遗漏翻译 key
- 所有交互触摸目标 ≥ 44×44px

### 4. 测试纪律

| 类型 | 何时执行 | 方法 |
|------|----------|------|
| **编译测试** | 每次代码变更后 | `npm run build` 零错误 |
| **单元验证** | 完成单个接口/组件时 | curl/httpie 请求接口验证返回值；浏览器检查组件渲染 |
| **集成验证** | 完成一个阶段后 | 启动前后端联调，验证数据流端到端 |
| **全栈验证** | 阶段九（T-9.x） | Docker Compose 启动全部容器，执行 20 项功能检查 + 6 项安全检查 |
| **回归检查** | 修复 bug 后 | 重新执行当前任务测试 + 前一个任务的关键测试，确认无回归 |

### 5. 错误处理决策树

```
遇到错误
├── 编译错误 → 阅读错误信息 → 定位文件和行号 → 修复 → 重新编译
├── 运行时错误 → 阅读控制台日志 → 定位异常 → 修复 → 重启验证
├── 测试不通过 → 对比预期与实际 → 定位差异原因 → 修复 → 重新测试
├── 依赖安装失败 → 检查包名和版本 → 尝试指定版本安装 → 换用替代包（最后手段）
├── 端口冲突 → 终止占用进程 → 或修改端口配置 → 重新启动
└── 数据库连接失败 → 检查 MySQL 运行状态 → 检查 DATABASE_URL → 检查用户权限
```

## 关键业务规则速查

以下是从 `requirements.md` 提取的关键规则，开发时务必对照：

### 单位转换
| 数据 | 输入可选单位 | 数据库存储单位 | 前端显示单位 | 转换公式 |
|------|-------------|---------------|-------------|----------|
| 月流量 | GB 或 TB | GB | TB（两位小数） | 1 TB = 1000 GB |
| 带宽 | Mbps 或 Gbps | Mbps | Gbps（两位小数） | 1 Gbps = 1000 Mbps |

### 价格
- 存储：数值（Float）+ 货币代码（3 位字母，ISO 4217）分开存储
- 显示：`22.00 USD`、`49.99 EUR`
- 排序：仅按数值排序，不做汇率转换，前端标注"按数值排序仅供参考"

### 排序循环
- 点击列头循环：默认 → 升序 → 降序 → 默认

### 分页
- 前端默认 50 条/页
- 后台默认 20 条/页

### 错误码
| 代码 | 含义 |

| 400 | 请求参数错误 |
| 401 | 未登录或 Token 过期 |

| 500 | 服务器内部错误 |
| 1001 | 用户名或密码错误 |
| 1002 | 登录失败次数过多 |
| 2001 | 产品不存在 |
